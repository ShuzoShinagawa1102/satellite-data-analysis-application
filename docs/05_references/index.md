## 2. 参照ポリシー（本仕様書における扱い）
### 2.1 優先順位
1. **法令・省庁公開情報**
2. **自治体公式公開情報**
3. **自治体事業の準公式紹介**
4. 民間記事・解説（本書では原則非採用）

### 2.2 参照の姿勢
- 本仕様書は「法的解釈書」ではない
- 法制度の最終確認は、常に最新の公式情報を参照する
- UI文言/業務ルールは、過度に法的断定表現を避ける

## 3. 用語集（本仕様書での意味）
## 3.1 ドメイン用語
### 盛土等
盛土、切土、土石の堆積等、本領域で監視対象として扱う行為・状態の総称。  
※法令上の定義・解釈とは厳密一致を要求しない（UI/運用上の総称として使用）。

### 危険・違法盛土（本仕様文脈）
行政上の監視対象となる可能性がある盛土等。  
本プロダクトでは、初期段階では「疑義候補」として扱い、違法性の最終判断は人が行う。

### 疑義地点
AI/ルール等で抽出された確認対象地点。違法確定ではない。

### トリアージ
複数の疑義地点に対し、現地確認等の優先順位を付ける行為。

### 現地確認
現場を確認し、写真・メモ・一次判断を記録する業務。

### 案件
継続的に追跡・管理すべき調査/確認/対応の管理単位。

### 証跡
画像、メモ、照合結果、状態遷移履歴、監査ログ等、後から判断経緯を説明するための記録。

## 3.2 システム用語
### 監査ログ
ユーザーやシステムによる重要操作の記録。変更前後・実行者・日時などを含む。

### 状態遷移
疑義地点または案件の状態変更ルール。勝手な飛び越しを防ぐための制御単位。

### 非同期ジョブ
取り込み・集計・レポート生成等、即時応答を必要としないバックグラウンド処理。

## 4. 仕様確定に用いた前提（明示）
本仕様は以下の前提のもとで確定している。

1. 技術スタックは React + Django 固定
2. 初期対象は自治体向けWebダッシュボード
3. AIは補助であり、最終判定責任を代替しない
4. 盛土監視に特化した単一サービスとして設計する
5. 提案/見積/スケジュールは本仕様書の対象外
